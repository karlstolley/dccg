Instant Course List

Well, you wrote a Ruby script to give us an HTML-formatted course list. You also changed the script so it put the question and response on the same line. That's the difference between print and puts: puts always adds a new line character at the end of a line.

Command line results from running create_html1.rb

[~/Projects/ruby/scripts]$ ruby create_html0.rb
What is the course number of your class? com531
What is the course title of your class? Web Development
Who teaches this class? Karl Stolley
What is day of the week does class meet? Wednesday
What time does class start? 6:25pm
What time does class end? 9:05pm
<table>
  <tr>
    <td>com531</td>
    <td>Web Development</td>
    <td>Karl Stolley</td>
    <td>Wednesday</td>
    <td>6:25pm</td>
    <td>9:05pm</td>
  </tr>
</table>
[~/Projects/ruby/scripts]$

Looks like we're done, right? You wrote a script so a user could add their courses from the command line and get a formatted list they could include in their own web page.

Well, you wrote a script that let the user add one course. What about the rest of them? Also this does not seem to be a very Ruby-like way of programming. The script works, but lacks a certain elegance; we end up repeating ourselves unnecessarily.

Let's streamline this script first, then we can write a script that captures all courses, not just one.

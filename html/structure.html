Structure

Body

The body of the webpage houses all of the content that is intended to be displayed to the public. The browser renders everything enclosed within the <body> and </body> tags so that it is viewable. All other tags that we learn about go inside the body of the document.

Tags Used Within the Body

Most of the following are pretty self-explanatory.

Section

The <section> tag is used to group like content together. Sections could be nested or stand alone. Usually, the sections contain headers and footers.

Header

The header comes at the beginning of each section and indicates that the information enclosed in the <header> and </header> is the header.

Article

The <article> tag is used for grouped information within sections. These are usually not nested unless they are nested within sections of other sections [aside: to see more about nesting, go to Nesting].

Paragraph

The <p> tag is probably the most common tag you will see in a document, simply because it defines the portion of text that should be displayed as a paragraph. It is possible for a very simple page to be composed simply of <p> </p> elements, however, it is very limited and it doesn't tell you much about the information enclosed. Also, when we start to style the page using CSS, other tags, such as section and article help better define parts of the webpage.

H1, H2, H3, H4, H5, and H6

The h tags are useful. In the example above, these were formed using these tags. Decreasing importance. In this page, the headings for each individual paragraph, for example section and article, use the h4 tag, while the heading for the entire page, HTML5, uses the h1 tag, indicated its hierarchal importance.

Navigation

The <nav> </nav>tag groups navigation links together. These links are usually presented as an unordered list (see below). [to see more about links go to Links and Images].

Unordered and Ordered Lists

Lists are used with surprising frequency when building webpages. The most common use is with navigation links but there are also other instances, such as the assumptions and learning objectives at the start of this tutorial. Lists help display information in an attractive and easy to read format, without the need for styling (though it's always looks better with a little style, see CSS).

There are two different tags for lists, depending on how you would like the information displayed: <ul> and <ol>. The <ul> tag is used when you do not want do put the information in any particular number order; it means unordered list. Depending on your styles or, if defined in styles, the browser default, a simple round bullet point precedes each item in the list. The <ol>, or ordered list, tag is used when you want the information to be displayed in order and using a number. The browser will automatically list the items in order 1, 2, 3, 4, etc, based on how you list it in the code.

In both cases, for each individual listed item you need to use the opening tag <li> and the closing tag </lil>.

Footer

The footer comes at the end of a section or the page with extra information.

Div tag

The <div> is a generic container that is used when none of the other tags appropriately define the information contained. They are used more often than not with CSS.

Using Script Files

By the End Of This Section You Should Be Able To...

  • Explain the two ways script tags can be used to include JavaScript on a web page
  • Create your own script files to include on your own web pages
  • Explain what a JavaScript library is, and know how to include them on your web page

So far, you've only used JavaScript using your in browser console. Obviously, this approach has some limitations. When you publish a web page, you want your JavaScript scripts deployed with it. We are going to take a break from JavaScript as a language and look into how to incorporate scripts with your web page.

There are two main ways to use JavaScript in your web pages. They both involve using the script tag. Let's take a look at a short example. 


<html>
<head>
<script type="text/javascript">
alert("Hello, world!");
</script>
</head>
<body>
This is my web page.
</body>
</html>

This is obviously a fairly trivial example, but it demonstrates the use of the script tag. You can put a script tag on your page, and any code that you put inside these tags will be interpreted as a script.

Usually we don't want to put JavaScript directly in our HTML pages, though. It's best to separate concerns, so we should keep our JavaScript in a separate file. To achieve the same effect as above, but without putting JavaScript directly into our HTML, we would first create a file called alert.js In this file we would put only our JavaScript code.

alert("Hello, world!");

Then, in our HTML, we would do the following. 


<html>
<head>
<script type="text/javascript" src="alert.js">
</script>
</head>
<body>
This is my web page.
</body>
</html>

What we did here was add a src attribute to the script tag that points to where our JavaScript file is. If we had put the file inside a scripts folder, we would write:

<script type="text/javascript" src="scripts/alert.js">

Besides separation of concerns and keeping JavaScript out of our HTML, this approach is good for reusing certain scripts across web pages. In this example, our script was only one line long. But imagine if we had a 50 line script that we want to run on every page of our web site. With the first approach we looked at, we'd have to copy and paste that script on every page. By using the src attribute, it's easy to include that 50 line script across many web pages by just using one script tag.

JavaScript Libraries

When you entered the code into your JavaScript console to change the background color of the page, you also relied on having access to a function called getRandomColor. I mentioned that this was a function that I wrote and that was include on the page.

Programmers often use code that other programmers write. If someone has already solved a problem, a lot of times it's easier to reuse their solution rather than to write a new one from scratch. getRandomColor was just one useful function I wrote, but sometimes programmers will package up a lot of variables and functions together and distribute them for other programmers to use. These are often referred to as frameworks or libraries. In the context of JavaScript, you usually use a library by downloading a .js file (like the alert.js file we created above), and then including that file using a script tag on any page where you want to use it.

We're going to spend the next chapter looking at one particular very popular JavaScript library called jQuery. The important thing to remember is just that if you hear someone talking about a JavaScript library, they are just referring to some code another programmer packaged up and distributed to help make your life easier.

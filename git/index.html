Why Git?

So you never make a mistake again

At first glance, Git seems overwhelming.

It is. Or rather, it tends to be.

But if you strip Git down to its essence, there are actually only a few things you do with any regularity.

Start Working

There are two ways to start working in Git: initialize a repository, or clone an existing one.

See What's Up

It's common practice to do the following three things to see what's going on in the repository and your work.

  1. What happened before: git log is ...
  2. What's happening now: git status is ...
  3. What's about to happen: git diff is ...

Work, Work, Work

You do work, you add files to be committed (git add) and then you commit them (git commit).

Share with the World

You share your changes with the rest of the world by running git push.

You grab changes from the world by running git pull, after specifying who "the world" is by running git remote add.

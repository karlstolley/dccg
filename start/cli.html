The Command Line

Learn it, loathe it, learn it more...loathe it less. Maybe even love it.

The first time you see it, the command-line interface (or more simply, the command line) seems like a throwback to a primitive era of computing.

It is.

But despite its appearance, the command line is a powerful means of interacting with a computer. And in certain cases, there are things that a command line can do that you simply can't do from within a fancy, contemporary graphical user interface (GUI)

Where am I?

The command line always opens putting you *somewhere* in the computer's file system. Most often this is in your home directory. On a Linux-like system, running the pwd command will output something like this: 

$ pwd
/home/username
$

Where username is actually your user name on the system, something like janesmith for Jane Smith.

Now, despite how boring that example is, it illustrates the command-line interface in action (and how it is often represented in instructional contexts and books). You have a command prompt, usually represented as $ (which you do not type), and you type a command (like pwd, or "print working directory") followed by the Return or Enter key. With many commands, the command line then offers some kind of output, /home/janesmith, followed by a new prompt, $, ready to accept another command. (However, oftentimes, the new prompt is not shown, as will be the style here.

What's here?

Once you've figured out where you are in the computer's file system, you can ask for a list of the directory's contents with the list command, ls. Running ls on Mac OS X in your home directory will output something like: 

$ ls
Desktop  Documents  Downloads  Library  Movies  Music  Pictures  Public

That's the listing of default directories for an OS X user account. If we wanted more details about the directory's contents, we can pass a flag, -l, to the ls command: 

$ ls -l
drwx------+  5 username  staff   170 Sep  4 14:42 Desktop
drwx------+  4 username  staff   136 Apr 23 22:19 Documents
drwx------+  4 username  staff   136 Apr 23 22:19 Downloads
drwx------+ 34 username  staff  1156 Sep  4 14:41 Library
drwx------+  3 username  staff   102 Apr 23 22:19 Movies
drwx------+  3 username  staff   102 Apr 23 22:19 Music
drwx------+  4 username  staff   136 Apr 23 22:19 Pictures
drwxr-xr-x+  5 username  staff   170 Apr 23 22:19 Public

Sometimes, rather than passing a flag, like -l, we pass an argument. To move, for example, from the working directory above into the Documents directory, we'd use the change directory command, cd, and give it Documents as an argument: 

$ cd Documents
$ pwd
/Users/username/Documents

What happens if you run cd by itself, with no arguments? Try it, and then run pwd again.

What Else?

The three commands we've looked at so far have told us our working directory, and the contents of the directory, as well as how to change directories.

Here are some other interesting commands to try; see if you can guess their purpose, based on their output: 

$ uptime
15:17  up 22:31, 3 users, load averages: 0.41 0.35 0.33
$ who
username  console  Oct 15 16:46
username  ttys000  Oct 16 12:32
username  ttys001  Oct 16 14:31
$ date
Tue Oct 16 15:19:05 CDT 2012
$ which date
/bin/date

You'll learn more about the command-line through an entire command-line program called Git.
